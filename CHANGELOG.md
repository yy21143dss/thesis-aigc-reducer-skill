# 更新日志

本文档记录 `thesis-aigc-reducer`（安装目录名建议仍为 `thesis-aigc-reducer`）各版本的主要变更。仓库在 GitHub 上为 [thesis-aigc-reducer-skill](https://github.com/yy21143dss/thesis-aigc-reducer-skill)。

## [未标记版本] - 2026-05-12

### 新增

- **改写后质量自查（语病检测机制）**：在 `SKILL.md` 中增加「改写后质量自查」流程，要求每次改写完成后、输出给用户之前，对结果逐句执行检查；涵盖语法与搭配、用词规范、语义准确、整体通读四个维度。若检查未通过，须在输出前修正，不得直接输出存在语病的文本。
- **LaTeX 流程衔接**：在 LaTeX 解析流程中增加「步骤五：质量自查」，与上述自查要求对齐。

### 强化

- **禁止以口语化降低 AIGC 率**：在通用约束与 AIGC 模式中明确「学术正式性」为最高优先级之一，列举典型口语与语气词禁用示例，并强调打破 AI 特征应通过句式与论述节奏调整，而非降低语言正式程度。
- **语法正确性与规范用词**：将「语法正确性」与学术正式性并列强调，说明典型语病类型、禁止生造词语与不规范搭配的原则，以及无法保证语法正确时应保留更接近原文的表达。
- **策略与示例同步**：`strategies.md` 强化「改写禁忌清单」与正式书面语原则；`examples.md` 中示例表述与当前策略一致；`README.md` 功能概览中补充对正式语体与改写方式的说明。

### 说明

- 本更新旨在减少「为降 AIGC 而牺牲学术规范」的改写倾向，通过输出前自检降低语病与不搭配表述进入终稿的概率。
- 检测结果仍取决于具体平台与版本，本 Skill 不保证任何检测或查重结果。
