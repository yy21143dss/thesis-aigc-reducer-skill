# 改写策略详细参考

本文件为 thesis-aigc-reducer skill 的改写策略参考。agent 在执行改写时应读取本文件并综合运用相关策略。

---

## AIGC降率策略

目标：让文本摆脱 AI 生成的规律性特征，使其更接近人类自然写作风格。

### 1. 句式重组

AI 生成的文本倾向于使用结构整齐、语法完美的句子。需要打破这种规律性。

**具体手法：**

- **长句拆短**：将一个包含多层修饰的长句拆为 2-3 个短句
  - 原句："随着深度学习技术的不断发展，越来越多的研究者开始将其应用于自然语言处理领域，并取得了显著的效果。"
  - 改写："深度学习技术发展迅速。不少研究者尝试将其引入自然语言处理领域，效果比较理想。"

- **短句合并**：将几个碎片短句合并为一个带有从句的复杂句
  - 原句："该方法效果好。它计算速度快。它适用范围广。"
  - 改写："该方法不仅在效果上有优势，计算速度和适用范围方面也表现得比较出色。"

- **倒装表达**：将结论/结果放在句首，原因/条件放在句尾
  - 原句："由于样本数量有限，实验结果可能存在一定偏差。"
  - 改写："实验结果可能有一定偏差，主要原因在于样本数量比较有限。"

- **插入语**：在句子中间插入补充说明，打破句子的"一气呵成"感
  - 原句："神经网络模型在图像分类任务中表现优异。"
  - 改写："神经网络模型——尤其是卷积网络——在图像分类这类任务中，表现算是比较优异的。"

### 2. 个人化表达注入

AI 文本通常缺乏个人视角，语气客观到"不像人话"。适度注入作者视角可以有效降低 AI 检测率。

**中文常用表达（按使用场景分类）：**

引出观点：
- "笔者认为"、"笔者发现"、"在笔者看来"
- "据笔者了解"、"从笔者的角度来看"

分析过渡：
- "经过分析发现"、"经过对比可以看出"
- "仔细观察后不难发现"、"对数据进一步分析后发现"

强调重要性：
- "值得注意的是"、"需要指出的是"、"这里有一个关键点"
- "不可忽视的是"、"特别需要关注的是"

概括总结：
- "从某种程度上说"、"总体来看"、"就目前的情况而言"
- "大致可以归纳为"、"简单来说"

**英文常用表达：**

- "It is worth noting that"、"From the author's perspective"
- "As can be observed from the results"、"Interestingly"
- "One possible explanation is that"、"It should be noted that"
- "To the best of our knowledge"、"As far as we are concerned"

**使用原则：**
- 每段使用 1-2 个即可，不要每句都加
- 分散在段首、段中、段尾，不要集中使用
- 选择与上下文语境匹配的表达

### 3. 句长节奏调整

AI 文本的典型特征是句长均匀，大部分句子在 20-30 字之间，几乎没有极短或极长的句子。

**调整策略：**

- 在段落中制造句长差异：混合使用 10 字以内的短句和 50 字以上的长句
- 偶尔使用不完整句或省略句（如"效果如何？见下文分析。"）
- 在列举或解释时，可以用一个短句收束（如"结果很明显。"）
- 避免连续 3 句以上句长相近

**示范：**
- AI 风格（句长均匀）："该模型在准确率方面达到了较高水平。同时在召回率方面也有不错的表现。此外模型的训练速度相比之前有了明显提升。"
- 人类风格（长短交替）："模型准确率不低。召回率方面，经过几轮调参之后，最终也达到了一个比较满意的水平——尽管中间走了不少弯路。训练速度倒是提升很明显。"

### 4. 段落结构打散

AI 几乎总是采用"总述 → 分点论述 → 总结"的三段式结构。

**打散策略：**

- **直接切入细节**：跳过总述句，直接从一个具体事实或数据开始
  - AI 风格："本节将从三个方面分析该方法的优势。首先..."
  - 改写："在实验中一个比较有意思的发现是..."

- **省略总结句**：段尾不必每次都"综上所述"，让读者自己归纳
  - 删掉或弱化"综上所述""总而言之""由此可见"等段尾总结

- **中途插入转折或疑问**：在论述中间加入"不过话说回来""当然这也不是没有问题"等转折

- **非对称分点**：如果有三个分论点，不必平均分配篇幅，可以对重点论点多写几句

### 5. 逻辑连接词弱化

AI 对"首先、其次、再次、最后、因此、然而、此外、同时"等连接词高度依赖。

**替换策略：**

| AI 偏爱的连接词 | 替代方式 |
|---|---|
| 首先 | 直接陈述第一点，不加连接词 |
| 其次 | "另外一点是""还有一个方面" |
| 再次 | "除此之外"、或直接省略 |
| 最后 | "还有就是""一个容易忽略的点是" |
| 因此 | "所以说""这也就意味着"、或直接省略 |
| 然而 | "不过""话虽如此""但实际上" |
| 此外 | "顺带一提""另外" |
| 同时 | "与此同时"、或直接用逗号连接 |

**原则：**
- 不必替换所有连接词，保留 30% 左右即可
- 有些地方直接删除连接词，用句意本身的逻辑关系来衔接
- 偶尔使用口语化的连接方式

## 传统降重策略

目标：在保持语义完全不变的前提下，彻底改变文本的表达方式，确保连续 13 字以上不与原文或他文重复。

### 1. 同义替换

替换非专业术语的词汇，保持语义不变。

**常用动词替换表：**

| 原词 | 可替换为 |
|---|---|
| 研究 | 探讨、探究、考察、分析 |
| 分析 | 剖析、解析、审视、考量 |
| 提出 | 指出、给出、提供、阐述 |
| 表明 | 显示、揭示、说明、体现 |
| 采用 | 运用、使用、借助、利用 |
| 实现 | 达成、完成、做到、达到 |
| 具有 | 拥有、带有、存在、包含 |
| 促进 | 推动、助力、带动、加速 |
| 导致 | 引起、造成、引发、带来 |
| 认为 | 指出、主张、强调、阐明 |

**常用名词替换表：**

| 原词 | 可替换为 |
|---|---|
| 方法 | 手段、途径、方式、路径 |
| 问题 | 难题、议题、困境、挑战 |
| 特点 | 特征、属性、特性、优势 |
| 领域 | 方向、范畴、层面、维度 |
| 效果 | 成效、结果、表现、成果 |
| 影响 | 作用、效应、冲击、波及 |
| 基础 | 根基、前提、根本、底层 |
| 趋势 | 走向、态势、动向、倾向 |

**常用形容词替换表：**

| 原词 | 可替换为 |
|---|---|
| 重要 | 关键、核心、不可或缺、举足轻重 |
| 明显 | 显著、突出、清晰、可观 |
| 广泛 | 普遍、大范围、全面、多方面 |
| 复杂 | 繁杂、多元、纷繁、错综 |
| 有效 | 高效、可靠、切实、行之有效 |

**注意事项：**
- 专业术语绝对不可替换
- 替换后需检查搭配是否通顺
- 同一段落中同一个替换词不要重复使用

### 2. 句式变换

**主动 ↔ 被动：**
- "本文提出了一种新方法" → "一种新方法在本文中被提出"/"本文中提出了一种新的方法"
- "研究者发现了该现象" → "该现象被研究者所发现"/"该现象在研究过程中得以发现"

**肯定 ↔ 双重否定：**
- "十分重要" → "不可忽视"/"不容小觑"
- "效果很好" → "效果并非不理想"/"效果不可谓不佳"
- "完全一致" → "并无差异"/"没有出现不一致的情况"

**陈述 ↔ 设问/反问：**
- "该方法效率高" → "该方法效率如何？从实验数据来看，效率是比较高的"
- "这个问题值得关注" → "为什么要关注这个问题？原因在于..."

**把字句 ↔ 被字句 ↔ 一般陈述：**
- "实验证明了该理论" → "该理论被实验所证明" → "通过实验，该理论获得了证明"

### 3. 拆句与合句

**长句拆分：**
- 原句："在当前大数据背景下，如何利用海量数据来训练高效的深度学习模型是学术界和工业界共同关注的核心问题。"
- 拆分："当前正处于大数据时代。如何借助海量数据训练出高效的深度学习模型，已经成为学术界与工业界都在关注的一个核心议题。"

**短句合并：**
- 原句："该算法速度快。该算法精度高。该算法易于部署。"
- 合并："该算法兼具速度快、精度高的优点，并且在实际部署方面也比较便捷。"

**句子成分重排：**
- 原句："在图像识别领域，卷积神经网络取得了突破性进展。"
- 重排："卷积神经网络在图像识别这一领域中，已经取得了突破性的进展。"

### 4. 语序调整

**因果倒置：**
- "因为数据量不足，所以模型泛化能力较差" → "模型泛化能力不太理想，究其原因，主要是数据量不够充足"

**条件后置：**
- "在学习率设置为0.01的条件下，模型收敛速度最快" → "模型收敛速度最快的情况，出现在学习率设为0.01时"

**时间/地点状语位移：**
- "近年来，自然语言处理技术发展迅速" → "自然语言处理技术在近些年发展得相当迅速"
- "在该实验中，准确率达到了95%" → "准确率在该实验中达到了95%的水平"

**定语/状语位置调整：**
- "具有高精度的检测算法" → "检测算法精度较高"/"一种精度比较高的检测算法"
